# Этап 5. Главная страница ЦБИ: структура для разработки

## 1. Задача этапа

Собрать главную страницу нового сайта ЦБИ в формате build-spec для разработки.

Цель:
- зафиксировать окончательный порядок секций
- определить состав каждой секции
- описать компоненты
- учесть desktop и mobile
- подготовить страницу к сборке в Cursor

Важно:
это не финальный дизайн-макет, а технически и смыслово точная структура главной страницы.

---

## 2. Роль главной страницы

Главная страница не должна быть длинным лендингом с бесконечным скроллом и повтором одной мысли.

Главная страница ЦБИ должна:
- быстро объяснить, кто вы
- показать, что здесь можно решить
- дать карту направлений
- показать доверительные опоры
- объяснить формат работы
- привести к следующему шагу

Главная = вход в систему сайта, а не склад всей информации.

## 4. Структура Header

### Задача
Дать чистую навигацию и не перегружать верх экрана.

### Состав
- логотип / название ЦБИ
- верхнее меню
- кнопка CTA

### Меню
- О ЦБИ
- Как мы работаем
- Решения
- Услуги
- Кейсы
- FAQ
- Контакты

### CTA в header
Обсудить задачу

### Для mobile
- без длинной верхней строки
- без дополнительного descriptor-текста
- только логотип + burger menu + CTA по необходимости
- никаких переносов в 2–3 строки

## 5. Структура Hero

## 5.1. Цель блока
Сразу дать статус, ясность и спокойное ощущение контроля.

## 5.2. Состав блока
- eyebrow
- H1
- subheading
- короткий supporting text
- 2 CTA
- опционально 1 визуальный блок / image area / abstract premium visual

## 5.3. Контент
### Eyebrow
Центр бизнес инвестиций

### H1
Инвестиционные решения в недвижимости России

### Subheading
Проверка, расчёт, структура сделки и сопровождение после покупки.

### Supporting text
Для инвесторов и предпринимателей, которым важны не обещания, а факты, риски и понятный процесс.

### CTA
- Посмотреть решения
- Обсудить задачу

## 5.4. UX-правила
- Hero должен быть визуально чистым
- никаких длинных полос текста сверху
- не ставить 4–5 смыслов в один экран
- не добавлять лишние бейджи
- не использовать длинный “манифест” в первом экране

## 5.5. Mobile
- H1 короче по ширине
- CTA в колонку
- supporting text компактнее
- визуал не должен ломать высоту экрана
- first viewport должен быть спокойным и коротким

## 16. Рекомендации для Cursor по реализации

### Архитектурно
- собирать как новый проект
- не брать старый лендинг за основу
- секции делать независимыми компонентами
- тексты хранить отдельно и чисто
- заложить масштабирование под внутренние страницы

### UX
- не перегружать hero
- повторять только 3 главных CTA
- не допускать длинных смысловых дублей
- следить за ритмом секций: смысл → доверие → действие

### Визуально
- премиальный минимализм
- много воздуха
- типографика важнее декора
- карточки строгие, ровные, без шумного оформления
- мобильная версия должна быть чище, чем desktop

---

## 17. Mobile-first требования

### Обязательно
- убрать любые длинные строки сверху
- hero в 1 колонку
- CTA stacked
- menu компактное
- карточки в 1 колонку
- process section вертикальная
- FAQ как accordion
- contact form короткая

### Нельзя
- 2–3 смысловых полосы над hero
- мелкие кнопки
- длинные карточки в два экрана
- большие отступы без смысла
- перегруженный header
