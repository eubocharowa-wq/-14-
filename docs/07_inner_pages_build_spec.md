# Этап 6. Внутренние страницы ЦБИ: build-spec для разработки

## 1. Задача этапа

Собрать структуру внутренних страниц нового сайта ЦБИ для разработки в Cursor.

Цель:
- зафиксировать архитектуру всех ключевых внутренних страниц
- определить состав блоков
- задать логику UX и mobile-first
- подготовить базу для кодинга и SEO-структуры

Важно:
это build-spec, а не просто описание “что хотелось бы видеть”.

---

## 2. Какие страницы входят в этап

1. О ЦБИ
2. Как мы работаем
3. Инвестиционные решения
4. Типовая страница направления
5. Услуги
6. Кейсы
7. FAQ
8. Контакты

---

## 3. Общие принципы для всех внутренних страниц

Каждая внутренняя страница должна:

1. Иметь ясный первый экран
2. Отвечать на один главный вопрос пользователя
3. Иметь логичный ритм блоков
4. Не повторять главную страницу
5. Завершаться спокойным CTA

### Для всех страниц действуют общие правила:
- короткий hero
- никакой длинной строки сверху
- один главный смысл на экран
- 1–2 CTA на страницу
- короткие абзацы
- строгая сетка
- mobile-first без перегруза

## 4. Страница “О ЦБИ”

## 4.1. Роль страницы
Укрепить доверие через личность Елены, философию ЦБИ и отличие от рынка.

## 4.2. Структура страницы

### Секция 1. Hero
- Eyebrow: О ЦБИ
- H1: Экспертный центр инвестиционных решений в недвижимости
- Subheading: ЦБИ — это не витрина объектов и не потоковая модель. Это работа через проверку, расчёт и понятный процесс.

### Секция 2. Кто такая Елена Бочарова
Цель:
- показать роль Елены
- вывести её из образа “просто специалиста по недвижимости”

Состав:
- заголовок
- фото / визуальный блок
- 2–3 абзаца
- тезисы о роли: куратор решения, стратег, человек, который удерживает качество входа

`FounderSection`

Цель:

  - проверка
  - расчёт
  - сопровождение
  - постсделка

Компонент:
`BrandModelSection`

Цель:
- показать отстройку спокойно, без агрессии

Формат:

Слева:

Справа:

Компонент:
`ComparisonSection`

### Секция 5. Принципы работы
- сначала документы и цифры
- право сказать “не идти дальше”
- результат после сделки важнее сделки
- уважение к капиталу и последствиям решения

Компонент:
`PrinciplesSection`

### Секция 6. CTA
- Обсудить задачу

Компонент:
`FinalCTA`

## 4.3. Mobile
- фото не должно давить высотой

## 12. SEO-логика внутренних страниц

Для разработки и дальнейшего продвижения важно:

### Каждая страница должна иметь:
- отдельный URL
- отдельный H1
- уникальный title и meta description
- уникальный смысловой интро-блок
- достаточную глубину текста
- нормальную заголовочную структуру H2/H3

### Особенно важны SEO-страницы:
- /resheniya
- /novostroiki
- /kommercheskaya-nedvizhimost
- /zemlya
- /stroitelstvo-domov
- /rekonstrukciya
- /proekty-razvitiya
- /uslugi
- /faq

## 13. Общие рекомендации для Cursor

### Архитектурно
- делать страницы через шаблоны и переиспользуемые секции
- page template для направлений обязателен
- тексты не хардкодить хаотично
- держать структуру готовой к расширению

### UX
- не пытаться сделать внутренние страницы “мини-лендингами”
- каждая страница должна раскрывать только свою роль
- CTA на странице — один главный, один вторичный максимум

### Mobile
- mobile-first обязателен
- меньше декоративных блоков
- больше вертикального ритма и воздуха
- без длинных строк в header/hero
